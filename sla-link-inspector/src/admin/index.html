Configurations

Configure when and how to notify about linked-issue SLA status. The Send SLA Alert to Linked Tickets panel sends this issue's SLA to linked tickets and notifies assignees.

SLA field (optional)

The app auto-detects an SLA custom field by name. If your site uses a different field name or language, set the custom field ID here so "Send SLA to linked tickets" works.

Find it in Jira: Project settings, Fields, then copy your SLA field ID. Jira Service Management SLA is also supported without this.

Notification type

Send notification as (you can enable more than one):

Post a comment on each linked ticket (about the parent's SLA).

Include @mentions in the comment so assignee/reporter/watchers get notified. Uncheck to post without pinging.

Option 1 – Incoming webhook (post to one channel):

Create at api.slack.com → Your app → Incoming Webhooks → Add New Webhook to Workspace.

Option 2 – Post to a channel (Channel ID):

⚠️ If both a webhook URL and Channel ID are set, the webhook takes precedence for channel posting.

Only for posting to a channel. The bot must be in that channel. In Slack: open the channel → channel name → View channel details for the ID.

Option 3 – DMs (no channel):

We send a direct message to each person in your Slack workspace by looking them up by email. They do not need to be in any channel—only in the same Slack workspace.

Requires the Bot token below. In your Slack app add scopes users:read.email and im:write. You can use DMs only (leave Channel ID blank) or use both channel + DMs.

Shared requirement — Bot token (Options 2 & 3)

From Slack: Your app → OAuth & Permissions → Bot User OAuth Token. Each Jira site saves its own token here.

Email (via webhook)

Send alert data to Zapier, Make.com, or similar so you can forward email yourself.

The webhook receives a JSON payload (issue keys, status, message, etc.). Build a Zap or scenario to send email.

Message templates

Two templates—pick the right one for how the message is sent:

Template Triggered by
Automated alert SLA status change: At Risk, Breached, or time-remaining warning (comments on linked tickets, optional Slack/email)
Manual send Clicking Send SLA to linked tickets in the issue panel—comment on linked tickets and matching Slack message

Automated alert template

Jira comment text when the parent SLA hits At Risk, Breached, or the time-remaining threshold. Also used for Slack/email for those events.

Variables:

  • {{issueKey}} — parent issue key (SLA ticket)
  • {{slaName}} — SLA field name
  • {{remainingTime}} — e.g. “9d left” or “2h overdue”
  • {{status}} — at risk / breached / X minutes remaining
  • {{assignee}} — @mentions from the linked ticket per your At Risk / Breached settings

Tip: Keep messages under 500 characters for best readability in Jira comments.

Manual send template

Comment on linked tickets when you use the panel action. Same wording goes to Slack if Slack is enabled for that flow.

Variables:

  • {{issueKey}} — parent issue whose SLA you’re relaying
  • {{slaStatus}} — At-Risk / Breached / In SLA range
  • {{expiryDate}} — formatted expiration date
  • {{assignee}} — @mentions on the linked ticket (assignee/reporter)

Example:

Linked ticket {{issueKey}} SLA is now {{slaStatus}}.

The date of expiration: {{expiryDate}}.

{{assignee}}

Tip: Keep messages under 500 characters for best readability in Jira comments.
