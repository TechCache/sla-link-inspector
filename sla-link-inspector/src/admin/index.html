Configurations

Configure when and how to notify about linked-issue SLA status. The Send SLA Alert to Linked Tickets panel sends this issue's SLA to linked tickets and notifies assignees.

SLA field (optional)

The app auto-detects an SLA custom field by name. If your site uses a different field name or language, set the custom field ID here so "Send SLA to linked tickets" works.

Find it in Jira: Project settings, Fields, then copy your SLA field ID. Jira Service Management SLA is also supported without this.

Custom message template

Used for alerts posted on each linked ticket when this (parent) ticket’s SLA goes at risk, breached, or 30 minutes remaining (Trigger conditions above). For the “Send SLA to linked tickets” comment, use the template below.

Variables:

  • {{issueKey}} – the parent issue key (the ticket that has the SLA)
  • {{slaName}} – SLA field name
  • {{remainingTime}} – e.g. “9d left” or “2h overdue”
  • {{status}} – at risk / breached / 30 minutes remaining
  • {{assignee}} – @mention(s) from the linked ticket (assignee, reporter, watchers, custom fields as configured)

Send SLA Alert to Linked Tickets message template

Used for the comment posted on linked tickets when you click “Send SLA to linked tickets” in the Send SLA Alert to Linked Tickets panel, and for the same message sent to Slack. Leave empty for the default.

Variables:

  • {{issueKey}} – the current issue key (the one whose SLA you’re relaying)
  • {{slaStatus}} – At-Risk / Breached / In SLA range
  • {{expiryDate}} – formatted expiration date
  • {{assignee}} – @mention(s) of the linked ticket’s assignee/reporter

Example:

Linked ticket {{issueKey}} SLA is now {{slaStatus}}.

The date of expiration: {{expiryDate}}.
